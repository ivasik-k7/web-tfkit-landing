# Contributing to tfkit

We love your input! We want to make contributing to tfkit as easy and transparent as possible.

## How to Contribute

### Report Bugs

- Use the bug report template
- Describe what you expected to happen
- Include steps to reproduce the issue

### Suggest Features

- Use the feature request template
- Explain why this feature would be useful
- Suggest how it should work

### Code Contributions

1. **Fork** the repo on GitHub
2. **Clone** your forked repository
3. **Create a branch** for your changes:
   ```bash
   git checkout -b feature/amazing-feature
   ```
4. **Make your changes**
5. **Test your changes**
   ```bash
   npm run test
   npm run build
   ```
6. **Commit your changes** with clear messages
7. **Push** to your branch
8. **Open a Pull Request**

## Development Setup

```bash
# Install dependencies
npm install

# Start development server
npm run dev

# Run tests
npm run test

# Build for production
npm run build
```

## Code Guidelines

- Use TypeScript for new code
- Follow the existing code style
- Add tests for new features
- Update documentation as needed
- Keep commits focused and descriptive

## Pull Request Process

1. Ensure your code passes all tests
2. Update documentation if needed
3. The PR will be reviewed by maintainers
4. Address any review feedback
5. PR will be merged once approved

## Questions?

- Open an issue for bug reports
- Start a discussion for questions
- Check existing issues before creating new ones
